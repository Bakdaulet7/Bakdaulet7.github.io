Python Data Wrangling for BI Developers

If you’re a Power BI developer looking to add Python to your toolkit, here’s where to start — and what it unlocks that DAX and M simply can’t do.
Published

October 22, 2024

I started in Power BI — DAX, M, the whole ecosystem. Python came later, but it fundamentally changed what I could build. Here’s a practical introduction for BI developers making the same shift.

Why Python alongside Power BI?

DAX is excellent for columnar calculations at query time. M is great for ETL within Power Query. But both have ceilings:

  • Statistical modeling beyond simple aggregates? Python.
  • NLP on text fields? Python.
  • API calls during data prep? Python (or a custom connector, but Python is faster to prototype).
  • Data quality scoring across millions of rows? Python.

The two ecosystems complement each other. I use Python for heavy lifting in data prep and modeling, then feed clean data into Power BI for visualization.

The core libraries

import pandas as pd       # DataFrames — your M equivalent
import numpy as np        # Numerical operations
import matplotlib.pyplot as plt  # Quick plots

If you’ve worked with Power Query tables, pandas DataFrames will feel familiar. Think of a DataFrame as a table you can manipulate with Python code.

A real wrangling example

Here’s something I do regularly — standardizing vendor data from multiple sources that use inconsistent naming:

import pandas as pd
import re

df = pd.read_csv('vendor_data.csv')

# Standardize company names
def clean_company_name(name):
    if pd.isna(name):
        return 'Unknown'
    # Remove legal suffixes, strip whitespace, title case
    name = re.sub(r'\b(LLC|Inc|Corp|Ltd|Co)\.?\b', '', str(name))
    return ' '.join(name.strip().split()).title()

df['company_clean'] = df['company_raw'].apply(clean_company_name)

# Standardize date columns
df['event_date'] = pd.to_datetime(df['event_date'], errors='coerce')
df['event_year'] = df['event_date'].dt.year
df['event_month'] = df['event_date'].dt.month

# Flag missing critical fields
df['has_missing'] = df[['company_clean', 'event_date', 'amount']].isna().any(axis=1)

print(df['has_missing'].value_counts())

The equivalent in M would be 3x longer and much harder to read.

Connecting Python output to Power BI

Once you have clean data in Python, there are a few ways to get it into Power BI:

  1. Export to CSV/Parquet and load in Power Query
  2. Python visuals (run Python directly in a Power BI report — limited but useful for custom charts)
  3. Write to a database that Power BI connects to (preferred for production)

For production pipelines, I use option 3 — Python scripts run on a schedule (via Azure Functions or a VM), write results to SQL Server or Azure Data Lake, and Power BI picks them up on refresh.

Where to go next

  • pandas documentation — it’s genuinely good: pandas.pydata.org
  • “Python for Data Analysis” by Wes McKinney — written by the creator of pandas
  • Kaggle’s free Python course — practical, short exercises

The learning curve from M to Python is real but shorter than you’d expect. Within a month of focused practice, you’ll find yourself reaching for Python first on complex data prep tasks.
