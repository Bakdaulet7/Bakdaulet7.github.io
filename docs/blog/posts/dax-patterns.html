5 DAX Patterns That Changed How I Build Reports

DAX
Power BI
Five DAX patterns I keep coming back to — not the basics, but the ones that solved real problems in production reports.
Published

January 20, 2026

After years of writing DAX professionally, certain patterns have become second nature. These five aren’t in most beginner tutorials, but they come up constantly in real work.

1. The SELECTEDVALUE safety pattern

SELECTEDVALUE returns the value of a column when the filter context contains exactly one distinct value — otherwise returns the alternate result. I use it constantly for dynamic titles and conditional measures.

SelectedRegion =
SELECTEDVALUE(Regions[RegionName], "All Regions")

This is safer than VALUES(...)[0] or FIRSTNONBLANK, which can error or behave unexpectedly with multi-select slicers.

2. Rolling N-period calculation

The classic rolling average pattern, but generalized for any period selection:

Rolling3MonthSales =
VAR MonthsToInclude = 3
VAR LastDate = MAX(Calendar[Date])
VAR FirstDate = EDATE(LastDate, -MonthsToInclude + 1)
RETURN
    CALCULATE(
        [TotalSales],
        Calendar[Date] >= EOMONTH(FirstDate, -1) + 1,
        Calendar[Date] <= LastDate,
        ALL(Calendar)
    )

Parameterizing MonthsToInclude and driving it from a slicer gives you a self-service rolling window without multiple hardcoded measures.

3. Disconnected table for what-if parameters

A disconnected table (no relationships to anything) combined with SELECTEDVALUE is the cleanest way to build what-if scenario inputs:

-- Disconnected table: DiscountRates with values 0, 0.05, 0.10, 0.15, 0.20
SelectedDiscount =
SELECTEDVALUE(DiscountRates[Rate], 0)

ProjectedRevenue =
[BaseRevenue] * (1 - [SelectedDiscount])

Users pick a discount rate from a slicer; the measure responds. No parameters, no what-if wizard quirks.

4. KEEPFILTERS for additive filtering

KEEPFILTERS is CALCULATE’s underappreciated sibling. Without it, CALCULATE replaces the existing filter context. With it, it intersects:

-- Without KEEPFILTERS: shows West Coast sales regardless of what's selected
WestCoastSales =
CALCULATE([TotalSales], Region[Area] = "West Coast")

-- With KEEPFILTERS: only shows West Coast IF West Coast is in the current filter context
WestCoastSales_Restricted =
CALCULATE([TotalSales], KEEPFILTERS(Region[Area] = "West Coast"))

I use KEEPFILTERS whenever I want a measure to “respect” slicer selections while also adding its own filter.

5. The virtual relationship pattern

Sometimes you need a filter to flow across tables that don’t have a physical relationship — or the physical relationship points the wrong way. TREATAS creates a virtual relationship at query time:

BudgetForSelectedProducts =
CALCULATE(
    SUM(Budget[Amount]),
    TREATAS(VALUES(Products[ProductID]), Budget[ProductID])
)

This takes the current filter context on Products[ProductID] and applies it as if it were a filter on Budget[ProductID], without a physical relationship. Extremely useful in composite models or when bridging fact tables.

DAX rewards pattern recognition. Once you’ve solved a problem the right way once, you’ll recognize the same shape in different contexts. These five patterns cover an enormous percentage of real reporting challenges.
